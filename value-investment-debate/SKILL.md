# 价值投资多角度辩论 (Value Investment Multi-Perspective Debate)

## 概述

本 Skill 启动一个结构化的多角色辩论流程，对投资标的或商业决策进行深度价值投资分析。

### 角色说明

| 角色类型 | 职责 | 调用方式 |
|----------|------|----------|
| **中立方（数据员）** | 搜集详尽数据、提供事实支撑、主持辩论、总结评分 | 本 Skill 内置 |
| **辩论角色** | 从各自视角提出投资论点，参与多轮辩论 | 动态加载，详见 roles/registry.md |

### 辩论角色配置

辩论角色由用户指定或使用预设组合。读取 `roles/registry.md` 获取：
- 可用角色列表（角色ID、名称、Skill路径）
- 预设组合（经典组合、三人行等）
- 自定义组合方式

**默认组合**：巴菲特 + 芒格（经典组合）

## 详细执行指南

### 第一阶段：数据搜集（中立方）

**数据质量标准**（必须严格执行）：

1. **时效性**：所有具有时效性的数据必须采用最新数据，禁止使用过期数据
2. **交叉验证**：所有数据必须经过至少2个独立来源交叉验证才可采用
3. **信源优先级**：
   - 第一优先：公司官方财报、公告（一手信源）
   - 第二优先：权威财经媒体（Bloomberg、Reuters、WSJ、财新等）
   - 第三优先：券商研究报告
   - 禁止：未经证实的社交媒体信息、匿名论坛消息

**搜集维度**：

1. **财务健康度**
   - 过去5-10年的收入、净利润、经营现金流
   - ROE、ROIC、毛利率、净利率趋势
   - 资产负债率、有息负债、现金储备
   - 自由现金流及资本支出

2. **估值分析**
   - 当前PE、PB、PS、EV/EBITDA
   - 历史估值区间百分位
   - 同行业可比公司估值对比
   - DCF估值敏感性分析

3. **竞争优势（护城河）**
   - 品牌护城河：品牌溢价、客户忠诚度
   - 成本护城河：规模经济、低成本结构
   - 网络效应：平台效应、用户粘性
   - 转换成本：客户迁移成本
   - 监管/牌照壁垒

4. **管理层评估**
   - 管理层背景与履历
   - 资本配置历史记录
   - 股东回报政策（分红、回购）
   - 诚信记录与透明度

5. **行业与宏观**
   - 行业增长前景与规模
   - 竞争格局与市场份额
   - 监管环境与政策风险
   - 宏观经济敏感性

**数据输出**：
- 内部传递使用结构化文本
- 每个数据点标注来源和验证状态
- 不确定的数据明确标注"待验证"
- **第一阶段数据包必须保存为md文件**（见下方文件保存规则）

### 第二阶段：多轮辩论

#### 辩论规则

**核心原则：数据优先，辩论基于完整数据。辩论中需要数据时，立即停下来搜集，验证后再继续。**

1. **发言顺序**：
   - 每轮由角色1先发言，角色2、角色3依次回应
   - 各方各有一次反驳机会

2. **论证要求**：
   - 必须基于已验证的数据包
   - 需要明确引用数据来源
   - 每方需提出至少1个核心投资论点 + 1个主要风险点

3. **实时数据补充机制（关键 - 执行顺序调整）**：

   **错误流程（禁止）**：
   ```
   角色1发言 → 角色2发言 → 交锋总结 → 补充数据 → 继续下一轮
   ```

   **正确流程（必须执行）**：
   ```
   角色1发言
      ↓（如提出需要数据支撑的论点）
   【暂停】立即搜集数据，验证后再继续
      ↓
   角色2发言
      ↓（如提出需要数据支撑的论点）
   【暂停】立即搜集数据，验证后再继续
      ↓
   角色3发言（如有）
      ↓（如提出需要数据支撑的论点）
   【暂停】立即搜集数据，验证后再继续
      ↓
   多方交锋
      ↓（如分歧需要数据验证）
   【暂停】立即搜集数据，验证后再继续
      ↓
   总结，进入下一轮或结束
   ```

   **具体执行**：
   - **角色1发言时**：如提出需要数据支撑的论点，中立方立即搜集验证，验证完再让角色2回应
   - **角色2发言时**：如提出需要数据支撑的论点，中立方立即搜集验证，验证完再让角色3回应（如有）
   - **角色3发言时**：如提出需要数据支撑的论点，中立方立即搜集验证，验证完再让各方反驳
   - **交锋时**：如出现需要数据验证的分歧，中立方立即搜集验证，验证完再继续交锋
   - **中立方主动识别**：不必等各方提出需求，自主判断数据缺口并立即补充

4. **数据补充标准**：
   - 同样需要交叉验证和最新数据
   - 数据未验证前，相关论点暂时搁置
   - 不带数据缺口进入下一轮辩论

#### 终止条件（高共识度时停止）

辩论持续进行，直到满足以下条件之一：

1. **高共识度**：双方投资判断一致（都买入/都观望/都放弃），且核心论据基本一致
2. **分歧已穷尽**：双方分歧点已充分讨论，继续辩论不会产生新洞察
3. **数据穷尽**：所有关键数据已搜集完毕，仍无法达成一致

**低共识度时继续**：如果共识度不高，自动进入下一轮，无需询问用户

#### 动态调用辩论角色

辩论时，根据 `roles/registry.md` 中指定的角色动态加载对应Skill。

**执行步骤**：
1. 读取 `roles/registry.md` 获取当前组合的参与者列表
2. 依次调用每个角色的Skill
3. 汇总各方观点后进入辩论交锋

**通用调用格式**：
```
use_skill: [角色Skill路径]

Prompt:
"作为[角色名]视角，请基于以下数据包分析[标的名称]的投资价值：

[数据包内容]

请输出：
1. 核心投资论点（买入/观望/放弃）
2. 关键风险点
3. 需要中立方补充的数据（如有）"
```

**示例调用**（假设用户选择"三人行"组合）：
```
角色1: buffett → 调用 buffett-perspective
角色2: munger  → 调用 munger-perspective
角色3: tang     → 调用 tang-chao-perspective
```

## 文件保存规则

**核心原则：每个阶段完成时立即保存，防止中途崩溃丢失数据。所有文件保留在技能目录下的辩论子目录中。**

### 个人发言文件保存规则（最优先）

**每人每次发言完成后立即保存为独立文件**：
- 文件名：`[标的名称]_[角色ID]_第N轮发言_YYYY-MM-DD.md`
- 内容：原封不动记录该角色本次发言的完整原文（Agent返回什么就保存什么，不做任何删减、摘要、格式化）
- 示例：`泡泡玛特_buffett_第1轮发言_2026-04-10.md`

**保存时机**：Agent返回发言结果后、中立方做任何分析之前，立即保存。

- 文件名：`[标的名称]_阶段1_数据包_YYYY-MM-DD.md`

## 注意事项

1. **数据优先原则（关键）**：辩论中需要数据时，**立即停下来搜集验证**，不在辩论后补充
2. **交叉验证**：所有关键数据必须经过至少2个独立来源验证
3. **时效性**：使用最新数据，禁止使用过期数据
4. **高共识停止**：只有共识度高时才停止辩论，低共识时继续
5. **保持中立**：中立方在辩论阶段不参与论证，只提供事实
6. **承认局限**：如果数据不足以支撑结论，在最终报告中明确说明
