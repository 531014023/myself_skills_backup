# 价值投资辩论 - 工作流程详细指南

## 数据搜集标准（严格执行）

### 时效性要求
- 所有具有时效性的数据必须采用最新数据
- 财务数据使用最新财报
- 估值数据使用当天/当周数据
- 禁止使用过期数据

### 交叉验证要求
- 所有关键数据必须经过至少2个独立来源交叉验证
- 标注数据来源和验证状态

### 信源优先级
1. **第一优先**：公司官方财报、公告（一手信源）
2. **第二优先**：权威财经媒体（Bloomberg、Reuters、WSJ、财新等）
3. **第三优先**：券商研究报告
4. **禁止**：未经证实的社交媒体信息、匿名论坛消息

## 快速启动模板

当用户触发辩论请求时，使用以下模板启动流程：

```markdown
我将为您启动「价值投资多角度辩论」分析流程。

**分析标的**：[标的名称]

流程将分为三个阶段：
1. 📊 **数据搜集**：我将作为中立方搜集详尽的数据资料（交叉验证+最新数据）
2. 💬 **多轮辩论**：调用巴菲特和芒格视角进行深度辩论（至少2轮，主动补充数据）
3. 📋 **评分总结**：评估论证质量，输出共识结论（高共识度时停止）

让我们开始第一阶段：数据搜集。
```

## 第一阶段详细执行步骤

### Step 1: 明确分析对象

与用户确认：
- 标的全称和股票代码（如有）
- 分析的时间范围
- 关注的特定问题（如有）

### Step 2: 启动数据搜集（多源交叉验证）

使用 web_search 工具从多个来源搜集信息：

```
搜索策略：
1. 公司官方财报（一手信源）
2. 权威财经媒体（Bloomberg、Reuters、东方财富、财新）
3. 券商研究报告
4. 行业数据

每个关键数据点必须从至少2个来源验证
```

### Step 3: 整理数据包（内部使用）

内部使用结构化文本（不保存md文件）：
```
数据包：[标的名称]
==================

[财务数据]
- 营收：X亿元（来源：公司2024年报+东方财富，已验证）
- 净利润：X亿元（来源：公司2024年报+Reuters，已验证）

[估值数据]
- 当前PE：X倍（来源：Bloomberg 2025-X-X，时效性：当天）

[待验证数据]
- XXX（来源单一，需补充验证）
```

## 第二阶段详细执行步骤

### 核心原则：数据优先，实时补充

**禁止流程**（辩论一轮结束后再补充数据）：
```
巴菲特发言 → 芒格发言 → 交锋总结 → 补充数据 → 继续下一轮
```

**必须执行流程**（辩论中需要数据时立即补充）：
```
巴菲特发言
   ↓（如提出需要数据支撑的论点）
【暂停】立即搜集验证 → 验证后再继续
   ↓
芒格发言
   ↓（如提出需要数据支撑的论点）
【暂停】立即搜集验证 → 验证后再继续
   ↓
交锋
   ↓（如分歧需数据验证）
【暂停】立即搜集验证 → 验证后再继续
   ↓
总结 → 共识度高？→ 结束 / 进入下一轮
```

### Step 1: 调用 Buffett Perspective

```
use_skill: buffett-perspective

请基于以下数据包，以巴菲特的视角分析 [公司名称] 的投资价值：

[数据包内容]

请输出以下四个部分：
1. **投资判断**：买入 / 观望 / 放弃
2. **核心投资论点**（如果是买入）：详细阐述为什么这是一个好投资
3. **核心不投资论点**（如果是放弃）：详细阐述为什么应该放弃
4. **关键风险点**：识别3个最重要的风险因素
5. **估值看法**：当前估值是否合理，内在价值区间估计
6. **需要中立方补充的数据**（如有）
```

### Step 2: 实时数据补充（如有需要）

如果巴菲特提出需要数据支撑的论点：
- **立即暂停辩论**
- 使用 web_search 搜集相关数据（交叉验证 + 最新时效）
- 验证后继续

### Step 3: 调用 Munger Perspective

```
use_skill: munger-perspective

请基于以下数据包，以查理·芒格的视角分析 [公司名称] 的投资价值：

[数据包内容]

请输出以下五个部分：
1. **投资判断**：买入 / 观望 / 放弃
2. **核心投资论点**：从多元思维模型角度阐述
3. **认知偏误提醒**：投资者可能在此标的上犯哪些认知错误
4. **反向思考**：如果这是一个糟糕的投资，原因会是什么
5. **对巴菲特观点的补充或反驳**
6. **需要中立方补充的数据**（如有）
```

### Step 4: 实时数据补充（如有需要）

如果芒格提出需要数据支撑的论点：
- **立即暂停辩论**
- 使用 web_search 搜集相关数据（交叉验证 + 最新时效）
- 验证后继续

### Step 5: 记录交锋点

在每一轮结束后，识别并记录：
1. 双方观点一致的领域（共识）
2. 双方观点不一致的领域（分歧）
3. 需要更多数据支撑才能判断的争议点

**注意**：如果交锋中出现需要数据验证的分歧，**立即暂停，搜集数据，验证后再继续**，不要等到下一轮。

## 第三阶段详细执行步骤

### 评分执行

为每个维度打分（1-10），并给出简要理由：

**数据使用准确性**：是否正确引用和解读数据？
**逻辑严谨性**：论证是否有逻辑漏洞？
**价值洞察深度**：是否有超越表面的深刻见解？
**风险识别完整性**：是否充分考虑了下行风险？
**表达清晰度**：论证是否清晰易懂？

### 共识提炼

共识应该满足：
- 双方都明确表达了相同或相似的观点
- 有数据或逻辑支撑

### 分歧记录

分歧应该明确记录：
- 分歧的具体内容
- 巴菲特方的立场和论据
- 芒格方的立场和论据
- 分歧的根本原因（可能是对同一数据的不同解读，或侧重不同的因素）

### 最终结论框架

```markdown
## 最终结论

### 投资建议
**结论**：[买入/观望/放弃]
**置信度**：[高/中/低]
**理由**：综合巴菲特和芒格的分析，...

### 关键决策因素
1. **支持投资的因素**：
   - [因素1]
   - [因素2]
2. **反对投资的因素**：
   - [因素1]
   - [因素2]

### 分歧处理
对于双方的分歧，我们倾向于[巴菲特/芒格/中立]的观点，因为...

### 风险提醒
- [风险1]：发生概率[高/中/低]，影响程度[大/中/小]
- [风险2]：...

### 建议仓位（如果是买入）
基于风险评估，建议配置 [X%] 的仓位

### 后续跟踪要点
- [指标1]：如果低于/高于X，需要重新评估
- [事件1]：关注[具体事件]的进展
- [时间节点]：[时间]后检查[具体事项]
```

## 特殊情况处理

### 数据不足时

**原则：数据优先，辩论基于完整数据。**

如果无法搜集到足够的数据：
1. **立即暂停辩论**，先搜集数据
2. 辩论过程中发现数据缺口，同样立即暂停搜集
3. 数据未验证前，不带数据缺口继续辩论
4. 在数据包中明确标注缺失的数据项
5. 在最终报告中降低置信度
6. 建议用户谨慎决策

### 双方观点完全一致时

如果双方观点高度一致：
1. 缩短辩论轮数（但仍至少2轮）
2. 重点记录双方的共识和共同担忧
3. 在最终结论中强化这一共识

### 双方观点完全对立时

如果双方观点完全对立：
1. 增加辩论轮数（3-4轮），深入探讨分歧
2. 尝试找到分歧的根本原因（对数据的理解不同？价值判断不同？）
3. 在最终结论中呈现两种可能性，让用户自己判断
