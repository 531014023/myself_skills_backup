> 本文档为 SKILL.md 提供渐进式披露的展开内容。AI 执行细节操作时自动读取。

## Ingest 完整步骤

**触发词：** 提供链接/文件/粘贴内容/文件夹

**输入类型及处理方式：**

| 输入类型 | 处理方式 |
|----------|----------|
| URL | WebFetch/smart-web-fetch 抓取 |
| 粘贴文本 | 直接解析 |
| 文件路径 | 读取并解析 |
| 文件夹路径 | 扫描并批量处理 |

**执行步骤：**

**Step 1. 检查是否已存在（必须）**
- 抓取前，先在 `raw/` 目录搜索该 URL（检查 frontmatter 的 `source` 字段）
- **搜索方法**（Windows PowerShell）：
  ```powershell
  cd ~/wiki/raw && Select-String -Pattern "source: https://..." -SimpleMatch
  ```
- 如果已存在：
  - 向用户报告：`该文章已保存于 raw/YYYY-MM-DD-标题.md`
  - 询问用户：是跳过 / 重新抓取 / 更新现有文件？
- **如果不存在：继续执行 Step 2**

**Step 2. 识别输入类型**
- URL → WebFetch/smart-web-fetch 抓取
- 文本 → 直接解析
- 文件夹 → 扫描所有 .md/.txt/.pdf 等文件（批量处理时每篇单独检查）

**Step 3. 解析内容**
- 提取标题、主要内容、关键术语
- 记录来源信息（URL、文件名、日期）
- 文件名格式: `YYYY-MM-DD-中文标题.md`
- 文件名中的中文标题必须和提取到的标题一致，不能改动

**Step 4. 存储到 raw/**
- 文件名: `YYYY-MM-DD-中文标题.md`（中文标题来自内容，保留所有标点）
- 添加 YAML frontmatter (id, source, date, tags, accessed)
- **id 必须与文件名完全一致（去掉 .md）**

**Step 5. 创建摘要层（必须）**
- **为每篇 raw 创建一篇摘要页**（`摘要/原文标题.md`）
- 摘要是整篇文章的浓缩，不是简单复制
- 包含核心观点、关键数据、重要结论
- frontmatter 的 `raw:` 指向该 raw 文件
- **关键原则：每篇 raw 必须先有摘要页**

### 用户粘贴内容时的处理方式

| 类型 | 处理方式 |
|------|---------|
| **所有内容** | ✅ 必须先存 raw → 创建摘要 → 提取 |

**原则**：无论外部文章还是用户原创内容，都必须先存 raw，保持规范一致性。

**Step 6. 自动提取知识层（必须）**
- 创建摘要后，**立即全面分析**该摘要内容
- **提取所有符合类型的内容**，不设"重要性"门槛：
  - **概念**：文中定义/提到的所有概念（如"坑口价"、"卖put"）
  - **实体**：文中提到的所有公司、人物、品牌（如"陕西煤业"、"段永平"）
  - **对比**：文中的对比分析（如"买put vs 卖put"）
  - **概览**：文中的体系/方法全面介绍
  - **综合**：跨领域思考或个人总结
- 检查是否已存在：
  - 若已存在 → 更新该页面（添加新 raw 关联）
  - 若不存在 → 创建新页面
- **原则**：单篇摘要即可提取，不等待多篇重复

**Step 7. 报告完成**
- 列出本次创建的所有摘要页
- 列出本次创建/更新的所有概念/实体/对比/概览页
- 标注各页面的 `raw:` 关联关系

**Step 8. 更新 index 和 log（必须）**

```
→ 更新 index.md...
✓ index.md 已更新（新增条目索引）
→ 更新 log.md...
✓ log.md 已记录（YYYY-MM-DD Ingest 完成：X个raw、X个摘要、X个概念、X个实体...）
```

**index.md 更新规则：**
- 每创建/更新一个 wiki 条目，同步更新 `index.md` 的相关分类索引
- 按类型分类记录：`摘要/`、`实体/`、`概念/`、`对比/`、`概览/`、`综合/`
- 格式：`[[标题]]`（不带日期）

**log.md 更新规则：**
- 每次操作必须追加一条日志
- 格式：
  ```markdown
  ## YYYY-MM-DD

  ### Ingest
  - 数量：X个raw、X个摘要、X个概念、X个实体...
  - 来源：URL/文件夹路径/粘贴
  - 新增条目：[[标题1]]...
  - 更新条目：[[标题2]]...
  ```
- 保持按日期倒序排列

## Ingest 提取可视化

```
raw/2026-01-01-文章A.md
raw/2026-01-02-文章B.md
raw/2026-01-03-文章C.md
       ↓【第一步：每篇 raw 创建一篇摘要】
摘要/文章A.md  ← 浓缩文章A的精华
摘要/文章B.md  ← 浓缩文章B的精华
摘要/文章C.md  ← 浓缩文章C的精华
       ↓【第二步：从摘要中提取知识】
概念/安全边际.md  ← 发现A、B、C都提到"安全边际"（多篇提炼 ✓）
概念/商业模式.md  ← 发现B、C都提到"商业模式"（多篇提炼 ✓）
概念/能力圈.md    ← 只有C提到，但概念重要（单篇提取 ✓）
实体/腾讯.md      ← 发现A、C都提到"腾讯"（多篇汇总 ✓）
实体/江南春.md    ← 只有A提到，但人物重要（单篇提取 ✓）
对比/酱香vs浓香.md ← 发现A、B有对比内容（多篇提炼）
概览/老唐投资体系.md ← 综合多篇的投资体系（多篇综合）
```

**关键检查点：**
- ✅ 每篇 raw 都有对应的摘要页？
- ✅ 是否经过摘要层（而非直接从 raw 拆分）？
- ✅ 概念/实体页：多篇提炼（推荐）或单篇提取（允许）？
- ✅ frontmatter 的 `raw:` 字段完整？

## Query 完整步骤

**触发词：** 提问

**⚠️ 关键检查点：** Query 开始前，确认用户是想「查 wiki」还是「让你搜外部资料回答」—— 如果 wiki 没有相关内容才搜外部。

**执行步骤：**

1. **理解问题**
   - 提取关键概念和术语
   - 识别问题类型（事实/解释/分析/建议）

2. **读取索引定位**（关键步骤）
   - 首先读取 `index.md` 索引文件
   - 根据索引文件的分类结构（摘要/实体/概念/对比/概览/综合）定位可能相关的条目
   - 不要直接用 grep 关键词搜索 wiki 文件夹

3. **读取相关条目并遍历内嵌引用**
   - 根据索引定位的结果，读取相关 wiki 条目内容
   - 沿着条目中的 `[[wikilinks]]` 继续深入阅读相关页面，将知识图谱串联起来
   - 综合多个条目及其引用内容回答

4. **补充来源**
   - 当 wiki 条目内容不足以回答时，引用 raw/ 中的原始素材
   - 提供可核查的引用

5. **标注不确定性**
   - confidence 等级 (high/medium/low)
   - 明确知识空白区域

6. **富维基触发**
   - 当答案产生了有价值的产物（比较、分析、联系）时，提示用户：「这似乎值得保留。把它归档为维基页面？」
   - 如果用户确认 → 先在 `raw/` 添加一条 AI 合成笔记（引用原对话） → 再执行完整 Ingest 流程（创建摘要 → 提取知识层 → 更新 index 和 log）

**Query 完成时的 log 更新：**
```
→ 更新 log.md...
✓ log.md 已记录（YYYY-MM-DD Query 完成：查询「XXX」）
```

## 富维基机制

**触发词：** 「存到维基」「这个值得记」「反馈到 wiki」

**判断标准：** 这个回答是否解决了一个以后还会问的问题？

**执行步骤：**

1. 将本次回答的完整内容写入 `raw/` 目录，创建一个 AI 合成笔记（frontmatter 的 `source` 字段注明对话来源）
2. 执行完整的 Ingest 流程：创建摘要 → 提取知识层（概念/实体/对比/概览/综合）→ 更新 index 和 log

**富维基输出示例：**

```
用户: 老唐的三板斧是什么？
AI: [综合回答，标注 confidence 和来源]

用户: 存到维基
AI: → 将 AI 合成笔记写入 raw/...
✓ raw/2026-04-16-富维基-老唐三板斧分析.md 已创建
→ 执行 Ingest 流程...
✓ wiki/摘要/富维基-老唐三板斧分析.md 已创建
✓ wiki/概念/至简三板斧.md 已更新（新增关联）
→ 更新 index.md...
✓ index.md 已更新
→ 更新 log.md...
✓ log.md 已记录（YYYY-MM-DD 富维基完成：新增[[老唐三板斧分析]]）
```

## Lint 完整步骤

**触发词：** 「维基体检」「lint」「检查维基」

**执行步骤：**

1. **检查断链**
   - 扫描所有 `[[wikilinks]]`
   - 验证目标文件在 wiki/ 目录存在

2. **发现矛盾**
   - 同一概念在不同条目的说法对比
   - 事实性错误检测
   - 标注矛盾程度（严重/轻微）

3. **识别知识空白**
   - 相关主题缺失检测
   - 过时信息标注（超过 30 天未更新）
   - 孤儿页面检测（无引用链的页面）

4. **输出报告**
   - 问题列表（类型、位置、建议修复）
   - 优先级排序

**执行频率：** 每两周一次，或积累 20+ 新素材后集中 lint。

**Lint 输出示例：**

```
两周后，用户: 维基体检
AI:
  ✓ 断链: 无
  ⚠ 矛盾: wiki/概念/至简三板斧.md 与 wiki/概念/安全边际.md
          对「估值标准」描述需统一
  ⚠ 孤儿: wiki/实体/古井贡.md 无引用
  ⚠ 过时: 商业模式.md 超过 30 天未更新
  建议: 确认估值标准；补充古井贡引用或合并
→ 更新 log.md...
✓ log.md 已记录（YYYY-MM-DD Lint 完成：发现3个问题，断链0个）
```

## 禁止编造规则

**绝对禁止的行为：**
1. **不得虚构 raw 原文中不存在的信息** - 所有内容必须能从 raw 中找到依据
2. **不得基于外部知识填充内容** - 即使你知道相关信息，如果 raw 中没提到，也不能写进 wiki
3. **不得自行补充缺失信息** - 遇到信息缺失时，必须停止并询问用户，而非猜测或编造
4. **不得更改原文含义** - 提取时要忠实于原文，不得曲解或过度解读

**遇到问题时的正确处理：**
```
情况：raw 文件内容异常/缺失/不清晰
❌ 错误：基于已有知识自行填充内容
✅ 正确：向用户汇报问题，询问如何处理

情况：需要提炼概念但信息不足
❌ 错误：从单篇 raw 强行创建概念页
✅ 正确：等待积累更多相关 raw 后再提炼
```

**质量承诺：**
- 所有 wiki 页面内容均可追溯到具体 raw 文件
- 每一条数据、每一个观点都有原文支撑
- 不确定的内容标注 `confidence: low` 并说明原因

  发现概念: "至简三板斧"、"All cash is equal" → ✓ 提取创建
  发现实体: "格雷厄姆" → ✓ 提取创建
  发现对比: "老唐法 vs 巴菲特法" → ✓ 提取创建
  发现概览: "老唐投资体系概览" → ✓ 提取创建
  ✓ 完成: 1个 raw → 1个摘要 + 3个概念 + 1个实体 + 1个对比 + 1个概览
  → 更新 index.md...
